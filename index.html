When you start or finish a coding session, do the following:

  1. When starting a coding session, you might want to check for pull requests on GitHub. Merge them so you aren't modifying old code.
  2. Open Git Shell
  3. cd <path of your working directory>
  4. Make sure you are on your own branch (the current branch is displayed in the prompt)
  5. git pull
  6. git merge origin/dev
  7. If there are conflicts, fix them by editing the files with a text editor
  8. git add -A
  9. git commit -m "<some commit message>"
  10. git push

At any time during these steps, you can use git status to see what's going on with git.

When you finish a feature, make a pull request:

  1. Go to https://github.com/GFUCABAM/statler
  2. Select your branch from the drop-down
  3. Click "Pull request"

Notes

If git push gives you a long message starting with warning: push.default is unset, enter git config --global push.default simple.

You can make aliases for git commands. See http://gitimmersion.com/lab_11.html.

You can use git log --graph --pretty=oneline to see a graph of git commits.
